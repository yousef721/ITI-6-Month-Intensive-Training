🎯 Project Requirements & Specifications

🏢 Client Requirements:

TechBooks Publishing's HR department needs a reliable system to manage their author database. The application must be built using industry-standard 3-tier architecture to ensure maintainability and future scalability.

📋 Required Core Features:
Feature 1: View All Authors - Display complete list of all authors with their contact information
Feature 2: Register New Author - Add new authors to the system when they sign publishing contracts
Feature 3: Remove Author - Delete author records when contracts are terminated or authors retire
⚠️ Technical Requirement from Development Team: Must use ADO.NET Disconnected Mode (DataAdapter with DataTable/DataSet). This is our company standard for data-heavy applications and allows for better scalability.

🏛️ Required 3-Tier Architecture Design

As per SoftSolutions Development Agency's coding standards, your application must be organized into three distinct layers. This ensures code reusability, maintainability, and allows the client to potentially upgrade to a web or desktop interface later without rewriting business logic.

1️⃣ Presentation Layer (User Interface)

Technology: Console Application

Responsibilities:

  • Display a professional menu system for HR staff
  • Collect user input (author details, choices, etc.)
  • Call methods from Business Logic Layer
  • Display results in a user-friendly format
  • CRITICAL: NO database code in this layer!
Design Tip: Think about the HR staff user experience. They need a clear, easy-to-use interface that doesn't require technical knowledge.
2️⃣ Business Logic Layer (BLL)

Components: Author Model + AuthorBLL Class

Responsibilities:

  • Define the Author business object/entity class
  • Coordinate between Presentation and Data Access layers
  • Transform data between database format (DataTable) and business objects (List<Author>)
  • Apply any business rules or validations
🔧 Required Mapping Functions (Critical!):

List<Author> ConvertDataTableToList(DataTable dt)
Purpose: Converts the entire DataTable from DAL into a list of Author objects

Author ConvertDataRowToAuthor(DataRow row)
Purpose: Converts a single DataRow into one Author object (used by the above method)
3️⃣ Data Access Layer (DAL)

Technology: ADO.NET (Disconnected Mode)

Responsibilities:

  • Manage all database connections and operations
  • Use SqlConnection, SqlCommand, and SqlDataAdapter
  • Execute SQL queries and return DataTables
  • Handle database exceptions properly
🔧 Required DAL Methods:

DataTable GetAllAuthors()
• Returns: DataTable containing all author records
• Uses: SqlDataAdapter.Fill() method

int InsertAuthor(string auId, string lname, string fname, ...)
• Returns: Number of rows affected (should be 1 if successful)
• Takes: All author properties as parameters
• Uses: Parameterized INSERT query

int DeleteAuthor(string auId)
• Returns: Number of rows affected (should be 1 if successful)
• Takes: Author ID to delete
• Uses: Parameterized DELETE query
